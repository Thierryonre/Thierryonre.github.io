◆ followLineBangBangDeadzone()

void Control::followLineBangBangDeadzone ( void )
inline

Follows a line using bang-bang control with deadzone.

The function continuously reads sensor values to determine the error between the left and right sensors. Based on this error, it adjusts the motor speeds to keep the buggy following the line. If the error is within the deadzone defined by config.bangBangDeadzone, the buggy maintains its speed. If the error exceeds the deadzone, the buggy adjusts the motor speeds accordingly to steer back onto the line.

◆ followLineBangBangProportional()

void Control::followLineBangBangProportional ( void )
inline

Follows a line using bang-bang with proportional control.

The function continuously reads sensor values to determine the error between the left and right sensors. Based on this error, it adjusts the motor speeds proportionally to the error magnitude, using the configured proportional gain defined by config.bangBangProportionalGain. If the error is negative, indicating that the robot is on the left side of the white line, it speeds up the left wheel and slows down the right wheel, and vice versa.

void Control::turnAround ( void )
inline

Turns the buggy around by 180 degrees.

This function retrieves the sensor values, stops the control algorithm, sets the motor speed and direction to make the robot turn around, waits for a certain period, and then waits until the white line is in the center of the sensors array before setting the motor direction forward again, resetting the control algoithm variables, and starting the control algorithm up again.
